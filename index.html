XVM-16 Lite + MIDI 仕様書

シンプル8bitバーチャルマシン - 命令削減 + MIDI音源版

コンセプト: 命令数を最小限に絞り、学習・実装が容易なシンプルバーチャルマシン。 音源はMIDI対応により本格的なサウンド表現が可能。256x256解像度、256色パレット、MIDI 16ch対応。

1. 基本仕様

1.1 CPU

  • ビット幅: 8bit CPU
  • アドレス空間: 16bit (64KB)
  • クロック: 4MHz
  • リフレッシュレート: 60Hz
  • サイクル/フレーム: 66,667 cycles

1.2 レジスタ（簡略化）

名前サイズ説明
A8bitアキュムレータ（演算結果）
X8bitXインデックスレジスタ
Y8bitYインデックスレジスタ
SP16bitスタックポインタ（初期値: 0x01FF）
PC16bitプログラムカウンタ（初期値: 0xE000）
F8bitフラグレジスタ
簡略化ポイント: B, C, Dレジスタを削除し、A, X, Yの3つのみに集約。

フラグレジスタ (F)

ビット名前説明
bit 0Z (Zero)演算結果が0
bit 1C (Carry)キャリー/ボロー発生
bit 2N (Negative)演算結果が負（bit7=1）
bit 3-7-未使用（常に0）

1.3 メモリマップ

アドレスサイズ説明
0x0000-0x00FF256BゼロページRAM
0x0100-0x01FF256Bスタック領域
0x0200-0x7FFF31.5KB汎用RAM
0x8000-0xFFFF32KBVRAM（グラフィック専用）
0xD000-0xD0FF256Bパレットメモリ（256色）
0xD100-0xD1FF256Bスプライト属性テーブル（64個）
0xD200-0xD2FF256BMIDIレジスタ
0xD300-0xD3FF256BI/Oレジスタ
0xE000-0xFFFF8KBROM（プログラム領域）

2. 命令セット（大幅削減版）

削減方針: 70以上あった命令を25命令に削減。 基本的な演算・転送・分岐のみに絞り、実装と学習を容易に。

2.1 データ転送（6命令）

命令オペランド説明サイクル
LDAimm/abs/zpg/idxA ← メモリ2-4
LDXimm/abs/zpgX ← メモリ2-4
LDYimm/abs/zpgY ← メモリ2-4
STAabs/zpg/idxメモリ ← A3-4
STXabs/zpgメモリ ← X3-4
STYabs/zpgメモリ ← Y3-4

2.2 演算（5命令）

命令オペランド説明フラグ
ADDimm/abs/zpgA ← A + メモリZ,C,N
SUBimm/abs/zpgA ← A - メモリZ,C,N
ANDimm/abs/zpgA ← A & メモリZ,N
ORAimm/abs/zpgA ← A | メモリZ,N
CMPimm/abs/zpgA - メモリ（比較のみ）Z,C,N

2.3 インクリメント/デクリメント（4命令）

命令オペランド説明
INCabs/zpgメモリ++
DECabs/zpgメモリ--
INX-X++
INY-Y++

2.4 分岐（4命令）

命令オペランド説明
JMPabs無条件ジャンプ
BEQrelZ=1なら分岐
BNErelZ=0なら分岐
BCSrelC=1なら分岐

2.5 サブルーチン（2命令）

命令オペランド説明
JSRabsサブルーチン呼び出し
RTS-サブルーチンから戻る

2.6 スタック（2命令）

命令オペランド説明
PHA-A をスタックにプッシュ
PLA-A をスタックからポップ

2.7 その他（2命令）

命令オペランド説明
NOP-何もしない
HLT-実行停止
合計25命令: 転送6 + 演算5 + Inc/Dec4 + 分岐4 + サブルーチン2 + スタック2 + その他2

2.8 アドレッシングモード

略称名前書式例説明

3. グラフィック仕様

3.1 ディスプレイ

  • 解像度: 256x256 ピクセル
  • 色数: 256色（8bit インデックスカラー）
  • VRAM容量: 65,536 bytes (256x256)
  • スプライト: 64個（16x16ピクセル）
  • レイヤー: BG + スプライト（2レイヤー）

3.2 カラーパレット

アドレス: 0xD000-0xD0FF (256色)

形式: RGB332（8bit）

RGB332形式:
RRR GGG BB
|||  |  ||
|||  |  ++-- 青 (0-3)  ×85 → 0-255
|||  +------ 緑 (0-7)  ×36 → 0-252
+++--------- 赤 (0-7)  ×36 → 0-252

3.3 スプライト仕様

スプライト数: 最大64個

サイズ: 16x16ピクセル（固定）

属性テーブル: 0xD100-0xD1FF (各スプライト4バイト)

スプライト属性 (4バイト/スプライト)

オフセット内容
+0X座標 (0-255)
+1Y座標 (0-255)
+2タイル番号 (0-255)
+3フラグ（bit0=有効, bit1=左右反転, bit2=上下反転）

4. MIDI音源仕様

MIDI対応の利点:
  • 標準MIDI音源（GM/GS/XG）対応で多彩な音色
  • 16チャンネル同時発音可能
  • ポルタメント、ピッチベンド、モジュレーション対応
  • エフェクト（リバーブ、コーラス）も利用可能

4.1 MIDIレジスタマップ

ベースアドレス: 0xD200-0xD2FF

アドレス内容値の範囲
0xD200MIDIコマンド送信0x00-0xFF
0xD201データバイト10x00-0xFF
0xD202データバイト20x00-0xFF
0xD203実行トリガー（書き込みで送信）任意
0xD204-0xD213チャンネル1-16 音色番号0-127（GM音色）
0xD214-0xD223チャンネル1-16 音量0-127
0xD224-0xD233チャンネル1-16 パン0-127（64=中央）

4.2 基本的なMIDI操作

音を鳴らす（Note On）

; チャンネル1、ノート60(C4)、ベロシティ100で発音
LDA #$90        ; Note On (ch1 = 0x90)
STA $D200
LDA #60         ; ノート番号(C4)
STA $D201
LDA #100        ; ベロシティ(音量)
STA $D202
LDA #$00        ; トリガー
STA $D203       ; 送信実行

音を止める（Note Off）

; チャンネル1、ノート60を停止
LDA #$80        ; Note Off (ch1 = 0x80)
STA $D200
LDA #60         ; ノート番号
STA $D201
LDA #64         ; リリースベロシティ
STA $D202
LDA #$00
STA $D203

音色変更（Program Change）

; チャンネル1を音色1（ピアノ）に変更
LDA #1
STA $D204       ; ch1音色レジスタに直接書き込み

6. サンプルプログラム

6.1 音階を鳴らす

; C4からC5までの音階を順に鳴らす

      LDX #60       ; ノート番号 = 60 (C4)
LOOP:
      LDA #$90      ; Note On, ch1
      STA $D200
      
      STX $D201     ; ノート番号
      
      LDA #100      ; ベロシティ
      STA $D202
      
      LDA #$00      ; 送信実行
      STA $D203
      
      ; 60フレーム(1秒)待機
      LDY #60
WAIT:
      LDA $D303     ; フレームカウンタ監視
      CMP $D303
      BEQ WAIT
      INY
      CPY #60
      BNE WAIT
      
      ; Note Off
      LDA #$80
      STA $D200
      STX $D201
      LDA #64
      STA $D202
      LDA #$00
      STA $D203
      
      INX           ; 次のノート
      CPX #73       ; C5まで?
      BNE LOOP
      
      HLT

6.2 和音を鳴らす

; C-E-Gのトライアド（Cメジャー）を鳴らす

      ; C4 (60)
      LDA #$90
      STA $D200
      LDA #60
      STA $D201
      LDA #100
      STA $D202
      LDA #$00
      STA $D203
      
      ; E4 (64)
      LDA #$90
      STA $D200
      LDA #64
      STA $D201
      LDA #100
      STA $D202
      LDA #$00
      STA $D203
      
      ; G4 (67)
      LDA #$90
      STA $D200
      LDA #67
      STA $D201
      LDA #100
      STA $D202
      LDA #$00
      STA $D203
      
      ; 2秒待機してから停止...
      HLT

9. 利点

9.1 実装の容易さ

  • 命令セット25個で実装時間が1/3に
  • レジスタ削減でデバッグが容易
  • アセンブラも簡単に作成可能

9.2 学習の容易さ

  • 覚える命令が少ない（25個）
  • 6502風のシンプルな設計
  • 初心者でも数時間で習得可能

9.3 音楽表現力の向上

  • MIDIで128音色が利用可能
  • 16チャンネル同時発音
  • ピアノ、ストリングス、ドラムなど本格的な音楽制作
  • 既存のMIDIシーケンサーとの連携
トレードオフ: 命令削減により一部の処理は複数命令の組み合わせが必要になるが、 シンプルさと学習コストの低下がそれを補って余りある利点となる。

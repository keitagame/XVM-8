01 概要 / OVERVIEW

RETRO-8はエミュレーター実装・低レベルプログラミングを第一目標に設計されたファンタジーコンソールです。命令数50以下・レジスタ直結のグラフィックシステム・シンプルなメモリマップを採用し、数百行のコードで動作するエミュレーターを書けるよう設計されています。

タイル形式

8×8 ピクセル、4bit/px（ニブル形式）。1タイル = 32バイト。各ピクセルはパレット番号0〜15。パレット番号0は透明色扱い（スプライト・BG共通）。

タイルマップの1エントリは1バイト：

[ bit7:FLIP_H | bit6:FLIP_V | bit5-4:palette_bank | bit3-0:tile_hi ]

次バイト: tile_lo → tile_id = tile_hi<<8 | tile_lo

※タイルマップは1エントリ2バイト（合計 32×32×2=2KB）

05 スプライト OAM

スプライト属性テーブル（OAM）は 0x0200–0x03FF。64体 × 8バイト。

オフセット名称説明
+0X_LOX座標下位8bit（-128〜383 で画面外もOK）
+1X_HIX座標上位（bit0のみ使用）
+2YY座標 (0〜255)
+3TILEタイルID下位8bit
+4ATTRbit7:FLIP_H / bit6:FLIP_V / bit5:PRIORITY / bit4:TILE_HI / bit3-0:palette_bank(0-3)
+5SIZE00=8×8 / 01=16×16 / 10=16×32 / 11=32×32
+6ANIM_CTRLbit7:ANIM_EN / bit6-4:frame_count(1-8) / bit3-0:anim_speed
+7ANIM_BASEアニメーション開始タイルID
アニメーション： ANIM_EN=1 にするとハードウェアが自動的に TILE を ANIM_BASE から frame_count 枚ループさせます。anim_speed は 0=1fr毎 〜 15=16fr毎 に更新。CPUゼロオーバーヘッドでアニメーション可能。
06 BG スクロール

BG0/BG1 はそれぞれ独立してピクセル単位でスクロール可能です。タイルマップは 32×32 タイル（256×256px）で、スクロールするとラップアラウンドします。

周波数計算

CH0/1/2 共通：f = CPU_CLOCK / (32 × (2048 - freq_reg))

例：A4 (440Hz) → freq_reg = 2048 - (4,000,000 / (32 × 440)) ≈ 1764

エミュレーター実装のヒント： 44100Hzでサンプルを生成する場合、1サンプルあたり CPU_CLOCK/44100 ≈ 90.7 サイクルごとに各チャンネルの波形を1ステップ進めると正確に再現できます。

11 ブートシーケンス / エミュレーター実装メモ

ブート手順

  1. 全メモリをゼロクリア
  2. パレット RAM をデフォルト32色で初期化
  3. SP = 0x01FF に設定
  4. PC = 0x8000（ROM バンク0 先頭）にジャンプ
  5. 割り込みを有効化

エミュレーターの最小ループ

while(running) {
  // 1. CPU フェッチ・デコード・実行
  cycles += exec(fetch(PC));

  // 2. PPU: HBLANK カウンタ更新
  if (++scanline >= 256) {
    scanline = 0;
    // VBLANK 割り込み
    trigger_irq(IRQ_VBLANK);
    render_frame();
  }

  // 3. APU: サウンドバッファ生成
  apu_step(cycles);
}
実装難易度： 命令デコードはバイト1の上位4bitでグループを確定 → 下位4bitで詳細命令を確定する2段階テーブルで実装可能。PPUはスキャンライン駆動でOAMを線形スキャンするだけ。APUは4チャンネルの独立カウンタ。エミュレーター全体は C 言語で約 1500〜2000 行で書けるよう設計されています。
